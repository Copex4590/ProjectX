# Project X 0.3.1-alpha.1 — Release Notes

**Release:** 0.3.1-alpha.1  
**Channel:** First Public Test  
**Date:** 2026-07-23

## Summary

First public test release after the SAVE-200…SAVE-205 release stabilization track. Focus: portable paths, clean shutdown, AIS reconnect backoff, HybridEngine async filesystem writer, SQLite WAL/batching, incremental map/radar updates, future-provider isolation, and unified version identity.

## Installation

### Linux (.deb)

```bash
sudo dpkg -i ProjectX.deb
sudo apt-get install -f
```

Launch **Project X** from the applications menu.  
Uninstall: Software Manager, or `sudo dpkg -r projectx`, or `ProjectX-uninstall.sh`.

### Linux (AppImage)

```bash
chmod +x ProjectX.AppImage
./ProjectX.AppImage
```

Data: `~/.local/share/projectx/`

### Windows

Run `ProjectX-Setup.exe` (after it is produced on a Windows build machine).  
Data: `%APPDATA%\Project X\`

Verify downloads with `SHA256SUMS`.

## Known issues

1. **Windows installer** may not be published until a native Windows build produces `ProjectX-Setup.exe`.
2. **Existing Linux binaries** in `release/linux/` may still embed older package metadata until rebuilt from this tag/branch.
3. **MarineTraffic / AISHub** shown as disabled (Coming Soon / not activatable).
4. **File → New Profile** disabled (Coming Soon).
5. **AIS reconnect** may take up to 60s after repeated failures.
6. **Logbook XLSX** may lag CSV until background worker finishes or logbook is opened.
7. **Legacy absolute data paths** are not auto-migrated — use env overrides if needed.
8. Alpha software — expect resets; not for unsupervised production.
