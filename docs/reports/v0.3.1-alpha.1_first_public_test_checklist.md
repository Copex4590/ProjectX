## A. Release identity (blockers if unchecked)

- [ ] Decide public version string: keep `0.3.1-alpha` **or** bump everywhere to `0.3.1-alpha.1` (must match About dialog).
- [ ] Align `src/version.py`, `release/manifest.json`, `website/releases.json`, Inno `MyAppVersion`, README, CHANGELOG.
- [ ] Set release build id via `PROJECTX_BUILD` (not `dev`) for packaged binaries.
- [ ] Record git revision (`git rev-parse HEAD`) in release notes / System Health export.
- [ ] Add CHANGELOG section for this public test.
- [ ] Update README version/status line.

## B. Packaging

### Linux

- [ ] Rebuild `.deb` and AppImage from current `release/0.3.1-alpha.1` HEAD (post SAVE-203).
- [ ] Refresh `release/linux/SHA256SUMS` and sync `website/downloads/linux/`.
- [ ] Install `.deb` on clean Ubuntu 22.04 / Mint 21+ VM; app appears in menu as **Project X**.
- [ ] Run AppImage (portable): executable bit, first launch creates `~/.local/share/projectx/`.
- [ ] Confirm writable data under user data dir (not inside install prefix).

### Windows

- [ ] Build PyInstaller + Inno → `release/windows/ProjectX-Setup.exe`.
- [ ] Align Inno version defines with chosen public version.
- [ ] Sync `website/downloads/windows/` + SHA256SUMS.
- [ ] Clean Windows 10/11 install → launch → uninstall smoke.
- [ ] Confirm `%APPDATA%\Project X\data` created on first run.

## C. Runtime smoke (every platform shipping)

- [ ] Clean startup (splash → main window).
- [ ] Clean shutdown (no hung threads / zombie AIS-catcher if used).
- [ ] No unhandled exception dialog on cold start with providers disabled.
- [ ] Enable AISStream only → connect → traffic → disable → offline.
- [ ] Enable RTL only (with AIS-Catcher) → traffic → disable.
- [ ] Hybrid (AISStream + RTL) enable/disable and reconnect after network kill.
- [ ] Offline mode (no providers): CPU idle, no reconnect spam.
