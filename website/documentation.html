Getting started

Project X is a desktop application for monitoring Danube vessel traffic. After installation, the first-run wizard guides you through language selection, observation point setup, AIS configuration, and optional camera setup.

  1. Download the installer for your platform from the Download page.
  2. Launch Project X and complete the first-run wizard.
  3. Open the Dashboard to verify AIS connection and live ship counts.
  4. Open the Live Map to view vessels on the interactive Leaflet map.
  5. Explore Timeline, Statistics, Alerts, and Camera features from the sidebar.

System requirements

Project X requires a modern desktop system with Qt WebEngine support for maps.

  • Windows: Windows 10 or later, 64-bit, 4 GB RAM minimum
  • Linux: Linux Mint or compatible distro with X11 or Wayland
  • Display: 1280×720 minimum resolution recommended
  • Network: Internet connection for AIS data and map tiles

Optional: mpv or vlc on PATH for external camera playback backends.

Windows installation

Run the Windows installer and follow the setup wizard. User configuration is stored under:

%APPDATA%\Project X\config\
%APPDATA%\Project X\data\

For development builds, use scripts\build_windows.bat on a native Windows system.

Linux installation

Which file should I download?

  • ProjectX.deb — recommended for Linux Mint, Ubuntu, and Debian. Installs menu integration and uninstalls cleanly.
  • ProjectX.AppImage — portable / advanced only if you do not want a system install. Does not add a menu entry automatically.
  • SHA256SUMS — optional integrity check after download.

Debian package (recommended for Linux Mint)

Double-click ProjectX.deb in your file manager to open GDebi, or install from a terminal:

sudo dpkg -i ProjectX.deb
sudo apt-get install -f

Launch Project X from the applications menu (short description: Professional Maritime Monitoring Platform). By default the installer also creates a desktop shortcut and can launch Project X when installation finishes. Uninstall with Software Manager or sudo dpkg -r projectx.

Portable AppImage (advanced)

Use this only when you need a single portable file and accept manual steps:

  1. Download ProjectX.AppImage.
  2. Right-click → Properties → Permissions → enable Allow executing file as program (or run chmod +x ProjectX.AppImage).
  3. Double-click to run, or execute ./ProjectX.AppImage from a terminal.

The AppImage does not install a menu shortcut. To remove it, delete the file. Optional menu integration requires a third-party tool such as AppImageLauncher — it is not built into this release.

User configuration is stored under ~/.local/share/projectx/ at runtime (not in the download folder).

Build from source

Clone the repository and install Python dependencies:

git clone https://github.com/Copex4590/ProjectX.git
cd ProjectX
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cd src
python3 main.py

See BUILD_WINDOWS.md and installer/README.md in the repository for packaging instructions.
